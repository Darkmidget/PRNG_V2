SECTION 1: UNDERSTANDING FINGERPRINT TEMPLATES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What is hex template data?
  - Binary representation of your fingerprint's unique patterns
  - Extracted from AS608 sensor as 128 bytes per scan
  - Stored as hexadecimal (hex) for easy file storage
  - Example: 03035F1D040142017C00000000...

Example template structure:
  Byte 0-1:   Header (0x03 0x03 = valid template)
  Byte 2-3:   Fingerprint quality/version info
  Byte 4-127: Ridge pattern data (minutiae points)

SECTION 3: HOW TO COLLECT MORE DATA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

QUICK START - RUN ONE OF THESE:

1️⃣  CONSISTENCY TEST (Validate same finger)
   Command: python test_consistency.py
   Purpose: Test finger stability and template variation
   Input:   Number of scans (2-10)
   Output:  Consistency report + CSV data
   Time:    5-15 minutes for 5 scans
   Use when: Want to validate a single finger


2️⃣  BULK COLLECTION (Collect from multiple fingers/people)
   Command: python bulk_collection.py
   Purpose: Build fingerprint database
   Input:   Person ID, finger name, number of scans
   Output:  bulk_collection.csv with all templates
   Time:    Variable (10 seconds per scan)
   Use when: Want to collect database of fingerprints


3️⃣  EXISTING EXPORT
   Command: python save_fingerprint.py --batch "export_all" ...
   Purpose: Export all enrolled fingerprints
   Use when: Just want raw template export


SECTION 4: COLLECTION PROTOCOL FOR BEST RESULTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PREPARATION (2 minutes)
☐ Clean sensor with dry cloth or lens paper
☐ Clean finger (dry, no lotion/moisture)
☐ Good lighting on sensor area
☐ Comfortable sitting position
☐ Test USB connection stability


PLACEMENT TECHNIQUE (Per scan)
☐ Position thumb centered on sensor
☐ Apply GENTLE, EVEN pressure (not pressing hard)
☐ Keep thumb stationary for 2-3 seconds
☐ Remove thumb SLOWLY (don't jerk)
☐ Wait for "Match Found" or beep confirmation


QUALITY CHECKLIST (After each scan)
☐ Template size is 128 bytes (not 0 or much larger)
☐ Hex data starts with "0303" (valid header)
☐ Hex has mix of all values (not all zeros/Fs)
☐ No "ERROR" messages in terminal

SECTION 6: TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Script shows "No match found"
Causes:  
  - Finger not on sensor when scanning
  - Insufficient pressure
  - Sensor not properly initialized
Solution: 
  1. Restart ESP32 (unplug/replug USB)
  2. Run "python test_consistency.py" again
  3. Pre-enroll fingerprint first


Problem: Template sizes vary widely (50, 100, 200 bytes)
Causes:
  - Different amounts of fingerprint on sensor each scan
  - Pressure variations
  - Partial scans
Solution:
  - Use consistent placement
  - Apply even pressure
  - Ensure full fingerprint contact


Problem: Hex data shows weird patterns (all zeros, all FFs)
Causes:
  - Sensor reading garbage data
  - Serial communication corruption
  - BLE/USB interference
Solution:
  1. Check USB cable is secure
  2. Restart terminal
  3. Verify COM5 is correct port

SECTION 8: DATA STORAGE & ORGANIZATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Recommended folder structure:
  fingerprints/
    ├── consistency_test.csv          (validation results)
    ├── templates.csv                 (individual exports)
    ├── bulk_collection.csv           (bulk data)
    ├── events.csv                    (system log)
    └── analysis/
        ├── similarity_report.txt
        ├── quality_metrics.csv
        └── variation_analysis.csv


CSV Format for bulk_collection.csv:
  Columns: timestamp, person_id, finger, scan_number, hex_data, size_bytes, matched_id
  
  Example:
  2026-04-02T04:50:15.123456+00:00,P001,Thumb,1,03035F1D040142017C00000...,128,1
  2026-04-02T04:50:25.234567+00:00,P001,Thumb,2,03035F1D040142017C00000...,128,1
  2026-04-02T04:51:30.345678+00:00,P001,Index,1,030361325401B8017600000...,128,4
